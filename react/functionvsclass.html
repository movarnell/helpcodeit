Function vs Class Components in React

How Should I Look at This?

These days React apps are mostly function components. They are easier to read, write, and test. They also perform better than class components.

However, class components are still used in some cases. For example, if you work for a company with older React applications they may be written in a Class based style.

What is a Function Component?

A function component is a JavaScript function that accepts props and returns a React element. It is a simple way to write a component that only contains a render method and doesn't have its own state. It is also known as a stateless component.

Modern React apps are almost exclusively written as function components. They are easier to read, write, and test. They also perform better than class components.

What is a Class Component?

A class component is a JavaScript class that extends React.Component. It is a more complex way to write a component that has its own state and lifecycle methods. It is also known as a stateful component.

It would be a good practice to focus on function components, but you may still need to work with class components in some cases.

What is the Difference Between a Function Component and a Class Component?

                    import React from 'react';

As you can see, the function based component is much easier to read and write. It is also easier to test. The class based component is more complex and has more code to write. It is also more difficult to test.

Why are Class Based Components Not Being Used?

Class based components are not being used because they are more complex and have more code to write. They are also more difficult to test.

While they aren't depricated yet, they will be soon. It is a good idea to focus on function based components.
