Import Issues in React

One of the most common issues in React development is importing components. This page will cover some of the most common issues with importing components in React.

React is a JavaScript library for building user interfaces. It is common in React development to need to import components that are reused. Below is an example of an import.

import React from 'react';

Imports are used to bring in components from other files. The import keyword is used to import a component. The from keyword is used to specify the file that the component is in. The file extension is not required. If the file extension is not specified, then the file extension will be assumed to be .js. If the file extension is specified, then the file extension will be used. The file extension is not required, but it is recommended to use it.

One of the more common issues with imports is forgetting to make your component exportable. If you do not export your component, then you will not be able to import it. Below is an example of a component that is not exportable.

function Welcome(props) {
    return 
Hello, {props.name}!
;
}

Notice that the component is not exportable. This means that it cannot be imported. To make the component exportable, you need to add the export keyword before the function. Below is an example of a component that is exportable.

export function Welcome(props) {
    return 
Hello, {props.name}!
;
}

Notice that the component is now exportable. This means that it can be imported. To import the component, you need to use the import keyword. Below is an example of importing the component.

import { Welcome } from './Welcome';

Now we will often use the default keyword when importing components. This is because we often want to import the default component. The default component is the component that is exported by default. The default component is the component that is exported if no other component is exported. Below is an example of a component that is exported by default.

export default function Welcome(props) {
    return 
Hello, {props.name}!
;
}

Notice that the component is now exported by default. This means that it can be imported. To import the component, you will do the same as we did when it was not the default component.

Importing a Component from a Specific Location

When you import a component, you can specify the location of the component. This is useful when you want to import a component from a specific location. Below is an example of importing a component from a specific location.

import { Welcome } from './components/Welcome';

When we are importing a component from the components folder we use "./components/Welcome". This means that we are importing the Welcome component from the components folder. If the component is in a folder outside of the components folder, then we would use "../components/Welcome". This means that we are importing the Welcome component from the components folder. If the component is in a folder outside of the components folder, then we would use "../components/Welcome".

How to See What The Relative Path Is?

If you can't quite figure out where the file is located you can right click on the file in vs code and click "Copy Relative Path". This will copy the relative path to the file. You can then paste the relative path into the import statement.

There is one thing you will need to do after this solution, you will need to make sure you take the unneeded parts of the path out, so if the relative path copies in as "user/username/documents/projectname/src/components/Welcome.js" you will need to remove the "user/username/documents/projectname/src/" part of the path. and just leave the "./components/Welcome.js" part of the path.

Often times if you're having issues with this if you work with the basic steps of trying different ways of using the "../" and "./" before you can figure out what is holding it back. Make sure to read any error messages closely
