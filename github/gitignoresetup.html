Using .gitignore to Prevent Files from Uploading to

This tutorial will show you how to use .gitignore to prevent files from uploading to GitHub.

This is useful for files that contain sensitive information, such as API keys, passwords, node_modules, and other information that you don't want to be public.

This tutorial assumes that you have already created a repository on GitHub and have cloned it to your local machine.

If you have not done this, please see the GitHub Basics tutorial.

If you have not installed Git on your local machine, please see the GitHub tutorial.

After making sure everything is installed, and if your up to this point in the course then it should have already been installed, you can continue with this tutorial.

First, you will need to create a .gitignore file in the root directory of your project.

This can be done by using the command touch .gitignore in the terminal, or by clicking the "Add File" button in vscode and typing ".gitignore" in the text box.

Next, you will need to open the .gitignore file and add the files that you want to ignore.

For example, if you want to ignore the node_modules folder, you would type 

node_modules/
in the .gitignore file.

If you want to ignore a file, you would type 

filename.extension
in the .gitignore file.

If you want to ignore a folder, you would type 

foldername/
in the .gitignore file.

If you want to ignore a file type, you would type 

*.extension
in the .gitignore file.

If you want to ignore a file type in a folder, you would type 

foldername/*.extension
in the .gitignore file.

If you want to ignore a file type in a folder and all subfolders, you would type 

foldername/**/*.extension
in the .gitignore file.

If you want to ignore a file type in a folder and all subfolders, you would type 

foldername/**/*.extension
in the .gitignore file.
