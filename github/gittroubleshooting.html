Getting Started with Troubleshooting

1. Git is not recognized as an internal or external command

Problem: When you type git into the terminal and see an error message saying that git is not recognized, it typically means that Git is not installed or not in your system's PATH.

Solution:

  1. First, check if Git is installed by typing git --version. If you get a version number, it means Git is installed.
  2. If Git is not installed, download and install it from the official website.
  3. Ensure that Git is added to your system's PATH. The method to do this varies depending on your operating system.

2. Permission denied (publickey) when trying to push or pull

Problem: You're trying to push or pull to/from a repository, but you get a Permission denied (publickey) error. This means your SSH key isn't being recognized by GitHub.

Solution:

  1. First, ensure you have an SSH key generated.
  2. If you don't have an SSH key, generate one.
  3. Once generated, add your public key to your GitHub account.
  4. Ensure your Git client is set up to use SSH.
git remote set-url origin git@github.com:USERNAME/REPO_NAME.git

3. Changes not staged for commit

Problem: You've modified files but when you run git status, you see a message "Changes not staged for commit".

Solution:

  1. To stage all changes, use the command:
git add .
  1. To stage specific files, use:
git add FILENAME
  1. Once staged, you can commit using:
git commit -m "Your commit message here"

4. Merge Conflicts

Problem: When merging branches, you sometimes encounter conflicts which Git can't automatically resolve.

Solution:

  1. First, identify the conflicted files with git status.
  2. Open each conflicted file in a text editor. You'll see conflict markers indicating the differing sections. Manually resolve each conflict by editing the file.
  3. After resolving all conflicts in a file, stage the file using: 
git add FILENAME
  1. Once all conflicts are resolved and staged, complete the merge with: 
git commit

5. Forgot to Pull Before Pushing

Problem: You made local changes and tried to push, but received an error because your local version is behind the remote.

Solution:

  1. Pull the latest changes from the remote:
git pull
  1. If there are merge conflicts, resolve them (as described above).
  2. Push your changes after resolving conflicts and committing:
git push

6. Accidentally Committed to the Wrong Branch

Problem: You made a commit, then realized you're on the wrong branch.

Solution:

  1. Switch to the correct branch:
git checkout CORRECT_BRANCH_NAME
  1. Cherry-pick the commit to the correct branch:
git cherry-pick COMMIT_HASH
  1. Return to the wrong branch and reset to the previous commit to undo the mistake: 
git checkout WRONG_BRANCH_NAME
git reset HEAD~1

7. Want to Undo the Last Commit

Problem: You made a commit and want to undo it.

Solution:

  1. To simply undo the last commit but keep the changes:
git reset HEAD~1
  1. To undo the last commit and discard the changes:
git reset --hard HEAD~1
