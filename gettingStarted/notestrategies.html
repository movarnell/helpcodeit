Taking Coding Notes - How to get setup:

This is a guide to help you get started with taking notes while you code. This is a great way to keep track of what you are learning and to be able to reference it later.

This guide will show you how to setup your VSCode to take notes and how to use it.

When getting started the first step is to get organized. I recommend adding a folder called either notes, or snippets to your workspace in vs code. That way you have a place to put the files. You will also want to make sure when you create a file to use to take notes you use the extension for the file type that represents the type of code you're using.

For example, if you are taking notes on HTML, you would want to use the .html extension. If you are taking notes on JavaScript, you would want to use the .js extension. This will help you to be able to reference your notes later and will provide the color coding so you can read the code easier.

How to Access Settings.json in VS Code

To access the settings.json file you will need to open the command palette. You can do this by pressing Ctrl + Shift + P on Windows or Cmd + Shift + P on Mac. Then type in settings.json and select the option that says 'Preferences: Open Settings (JSON)'. This will open the settings.json file. You can then add the settings for the extension to the file. You will only want to edit the existing settings, so scroll down till you see the settings specific to Comment Anchors.

Info on the Extensions

This extension allows you to add anchors to your comments. This is useful for taking notes because you can add an anchor to a comment and then reference it later. It keeps a list of all the 'Tagged' anchors you set. Read the page for the extension for more information on tagging. If you want to edit the settings to where the entire comment is highlighted, or if you want to change the color that shows up for a particular tag you will need to access the settings.json file for vs code.

Customizing Your Comments

Example


    "commentAnchors.tags": {
    "INFO": {
    iconColor is the color of the 
    icon to the left of the comment
    "iconColor": "#FFFFFF",
    Hightlight color is the color of 
    the tag in the comment
    "highlightColor": "#FFFFFF",
    "scope": "file",
    StyleComment when true highlights 
    the whole comment
    "styleComment": true
    },
}

Better Comments is an extension that allows you to add different types of comments. This is useful for taking notes because you can add different types of comments that are color coded much like the Comment Anchors extension. This is useful for adding a visual representation of what the comment is about. For example, if you are taking notes on a particular function you could add a comment with the emoji of a function. This will help you to be able to quickly find the notes you are looking for. You can also customize the colors of the different types of comments. You can also customize the color of the emojis. If you want to edit the settings you will need to access the settings.json file for vs code.

Polacode is an extension that allows you to take screenshots of your code. This is useful for taking notes because you can take a screenshot of your code and then add it to your notes. This will help you to be able to quickly find the notes you are looking for. You can also customize the colors of the different types of comments. You can also customize the color of the emojis. If you want to edit the settings you will need to access the settings.json file for vs code. You can copy the text from the screenshot and paste it into your notes or share it with anyone.

Now You Are Ready to Take Notes

With a folder for your notes, saved in files with the correct file extension type, and using the vscode extensions you are ready to start taking notes. Make sure to comment what the code is doing and what you are learning. This will help you to be able to reference your notes later. You can also add emojis to your comments to help you to be able to quickly find the notes you are looking for. You can also add screenshots of your code to your notes. This will help you to be able to quickly find the notes you are looking for. You can also customize the colors of the different types of comments. You can also customize the color of the emojis. If you want to edit the settings you will need to access the settings.json file for vs code. You can copy the text from the screenshot and paste it into your notes or share it with anyone.

Also make sure you use a file name when creating the code that tells you what the note is about. Camel case can make them easier to read. Use your own style, as these are something you will reference and don't have to adhere to any particular style guide.
